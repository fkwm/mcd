# mcd / mmcd

日毎のメモディレクトリを作成・移動するシェル関数です。

## 動作環境

Linux上の bash 、zsh 、sh(busybox) で動作するよう調整しています。

## インストール

保存した `mcd.sh` を `source` コマンドで読み込むと各関数が使用可能になります。
ログイン時に自動で読み込むには、使用するシェルに応じて
`~/.bashrc`（bash）や `~/.profile`（sh系）などに以下を記載してください。

```sh
source /path/to/mcd.sh
```

併せて必要に応じて環境変数を設定してください（後述）。

## mcd

### 概要

日付（YYYYMMDD形式）のディレクトリを作成し、そのディレクトリに移動します。
ディレクトリが存在しない場合は自動的に作成され、`memo.txt` ファイルも初期化されます。

### 使用方法

```sh
mcd [オプション]
```

| 使用例 | 動作 |
|---|---|
| `mcd` | 今日の日付のディレクトリに移動（例: `$HOME/tmp/20260102`） |
| `mcd -1` | 1つ前の記録ディレクトリに移動 |
| `mcd -3` | 3つ前の記録ディレクトリに移動 |
| `mcd +1` | 1つ後の記録ディレクトリに移動 |
| `mcd -h` | ヘルプを表示 |

#### 引数なし（今日）

```sh
$ mcd
```

`$MCD_NOTE_DIR_BASE`（デフォルト: `$HOME/tmp`）配下に今日の日付ディレクトリを作成して移動します。
ディレクトリが存在しない場合は作成され、`memo.txt` も初期化されます。

#### N項目前の記録に移動（`-N`）

```sh
$ mcd -1
```

カレントディレクトリを基準に、N個前の記録ディレクトリに移動します（探索モード）。
ディレクトリ作成・`memo.txt` 作成・テンプレート展開は行いません。

8桁数字で始まるディレクトリを降順ソートして探索します。
例: `20260105 > 20260104.test > 20260104 > 20260103`

#### N項目後の記録に移動（`+N`）

```sh
$ mcd +1
```

カレントディレクトリを基準に、N個後の記録ディレクトリに移動します（探索モード）。
詳細ルールは `-N` と同様です。

### 環境変数

| 変数名 | 説明 | デフォルト |
|---|---|---|
| `MCD_NOTE_DIR_BASE` | メモディレクトリのベースとなるディレクトリ | `$HOME/tmp` |
| `MCD_TEMPLATE_DIR` | テンプレートディレクトリのパス。設定されている場合、新規作成時にそのディレクトリ内のファイル・ディレクトリをコピーします。存在しない場合は警告を表示してデフォルトの `memo.txt` を作成します。 | 未設定 |

### エイリアス例

```sh
alias t="mcd"
alias tt="mcd -1"
alias tp="mcd -1"
alias tn="mcd +1"
```

#### エイリアス自動設定（`_mcd_init_aliases`）

`_mcd_init_aliases` を使うと、よく使うエイリアスをまとめて設定できます。
`~/.bashrc` や `~/.profile` などに以下を記載してください。

```sh
# デフォルトプレフィックス "t" でエイリアスを設定
eval `_mcd_init_aliases`

# プレフィックスを変更する場合（例: "m"）
eval `_mcd_init_aliases m`
```

プレフィックスを `t` にした場合、以下のエイリアスが設定されます。

| エイリアス | コマンド | 動作 |
|---|---|---|
| `t` | `mcd` | 今日の日付のディレクトリに移動 |
| `tt` | `mcd -1` | 1つ前の記録ディレクトリに移動 |
| `tp` | `mcd -1` | 1つ前の記録ディレクトリに移動（p=previous） |
| `tn` | `mcd +1` | 1つ後の記録ディレクトリに移動（n=next） |
| `tk` | `mcd -1` | 1つ前の記録ディレクトリに移動（vim: k=上） |
| `tj` | `mcd +1` | 1つ後の記録ディレクトリに移動（vim: j=下） |

## mmcd

### 概要

`mcd` のラッパー関数です。カレントディレクトリ付近の `tmp` ディレクトリを自動検出し、
`MCD_NOTE_DIR_BASE` を設定した上で `mcd` を実行します。

プロジェクトごとのメモを `./tmp/` 配下にまとめたい場合に便利です。

### 前提条件

- カレントディレクトリが `$HOME` 配下であること（それ以外はエラーで終了）

### 動作

1. `$PWD/tmp` が存在すれば、それを `MCD_NOTE_DIR_BASE` として使用
2. なければ `$PWD/../tmp`、`$PWD/../../tmp` ... と `$HOME` まで遡って探索
3. どこにも見つからない場合、`$PWD/tmp` の作成を確認してから実行

```
例: $PWD = $HOME/work/project の場合の探索順
  1. $HOME/work/project/tmp
  2. $HOME/work/tmp
  3. $HOME/tmp
  → 見つからない場合: "$HOME/work/project/tmp を作成しますか？"
```

### 使用方法

引数・オプションは `mcd` と完全に共通です。

```sh
mmcd        # 今日の日付のディレクトリに移動
mmcd -1     # 1つ前の記録ディレクトリに移動
mmcd +1     # 1つ後の記録ディレクトリに移動
mmcd -h     # ヘルプを表示（mcd のヘルプ）
```

## 開発者向け

### テスト環境（test/Makefile）

`test/Makefile` は busybox コンテナを使った動作確認用の環境です。
通常の使用では不要です。

```sh
cd test

make shell             # コンテナに入って手動確認
make shell ENGINE=podman  # Podman を使用する場合
make clean             # コンテナを削除
```

コンテナ内では `../src` が `/app/src` にマウントされています。
`source` で読み込むことで `mcd` / `mmcd` 等、mcd.sh の各関数を実行できます。

```sh
# コンテナ内での操作例
source /app/src/mcd.sh
mcd --help
```
